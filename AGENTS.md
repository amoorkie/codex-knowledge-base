# AGENTS.md

Этот репозиторий предназначен как база опорных источников для Codex-агента.

## Как использовать

1. Сначала смотри `index.yaml`, чтобы найти релевантные материалы по стеку и теме.
2. Если есть выбор, сначала открывай записи с `type: official` и `type: spec`.
3. Используй `.md` подборки как кураторский слой: они объясняют, зачем нужен источник.
4. Учитывай поле `verified`. Если тема быстро меняется, не полагайся на старую дату проверки без дополнительной верификации.

## Приоритет источников

- `official` - официальный docs/source of truth.
- `spec` - спецификации и стандарты.
- `reference` - канонические reference-материалы и де-факто стандарты.
- `community` - сильные community-источники, использовать после official/reference.

## Правила для агента

- Не воспринимай любую ссылку как абсолютную истину. Сравнивай источник, дату и контекст задачи.
- Для high-stakes тем перепроверяй информацию по первичному источнику.
- Если в подборке есть и official, и community, сперва используй official.
- Не цитируй длинные фрагменты источников; храни и используй краткие аннотации.

## Как расширять базу

- Добавляй только материалы с понятной практической ценностью.
- Избегай дубликатов.
- Если источник узкоспециализированный, помечай это в `why` и `topics`.
- Если материал устарел или появился более сильный аналог, обновляй существующую запись, а не плодить новые.
