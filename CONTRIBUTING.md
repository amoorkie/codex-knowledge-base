# Contributing

Этот репозиторий хранит curated knowledge base для Codex-агента и команды.

## Что добавлять

- официальную документацию;
- спецификации;
- сильные reference-материалы;
- полезные community-источники только с явным объяснением, почему они здесь;
- best practices, которые реально помогают в работе.

## Что не добавлять

- случайные подборки ссылок без объяснения;
- устаревшие статьи без причины хранить их дальше;
- дублирующие источники без новой ценности;
- длинные пересказы чужой документации вместо короткой аннотации и ссылки.

## Формат

1. Добавь или обнови профильный `.md` файл.
2. Для каждой записи укажи:
   - `url`
   - `type`
   - `why`
   - `topics`
   - `verified`
3. Обнови `index.yaml`.
4. Для записи в `index.yaml` обязательно задай:
   - `priority`
   - `scope`
   - `stability`
   - `trust`

## Проверки

Перед коммитом запусти:

```powershell
python scripts/validate_index.py --strict
python scripts/check_links.py
```

## Кураторские правила

- Приоритет у `official` и `spec`.
- `community` используется только как вспомогательный слой.
- Если тема быстро меняется, ставь свежую дату `verified`.
- Если ссылка больше не работает, сначала замени ее на актуальную official/reference страницу, а не просто удаляй тему целиком.

## Pull request checklist

- Обновлены соответствующие `.md` файлы.
- Обновлен `index.yaml`.
- Проставлены metadata-поля.
- Локальные проверки прошли.
- Вклад действительно улучшает базу, а не увеличивает шум.
