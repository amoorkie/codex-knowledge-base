# Codex Knowledge Base

Кураторский репозиторий ссылок, практик и опорных материалов для Codex-агента и команды.

Цель репозитория:

- хранить сильные, проверяемые источники по популярным стекам и технологиям;
- отделять официальную документацию от community-материалов и best practices;
- давать агенту понятный вход: что читать первым, чему доверять больше, что проверять по дате.

## Принципы

- Не складывать все подряд. Лучше меньше ссылок, но выше качество.
- Для каждой ссылки фиксировать, зачем она нужна.
- Приоритет официальным источникам и каноническим reference.
- Быстро меняющиеся материалы помечать датой проверки.
- Не копировать большие фрагменты чужого контента, а хранить ссылки и краткие аннотации.

## Структура

- `AGENTS.md` - инструкция для агента, как использовать репозиторий.
- `TEMPLATE.md` - шаблон для добавления новых подборок.
- `index.yaml` - машиночитаемый индекс.
- `frontend/` - фронтенд-стеки и UI-практики.
- `backend/` - серверные технологии и API-практики.
- `community/` - сильные community-источники с кураторскими выжимками.
- `data/` - data engineering, orchestration, analytics и MLOps.
- `databases/` - SQL/NoSQL, производительность, эксплуатация.
- `devops/` - контейнеры, CI/CD, инфраструктура, observability.
- `testing/` - тестовые инструменты и подходы.
- `security/` - secure coding, verification standards, cheat sheets.
- `ai/` - материалы по OpenAI, LLM app development, evals.

## Как оформлять материалы

Для каждой записи фиксируй:

- `title` - короткое название.
- `url` - ссылка на источник.
- `type` - `official`, `reference`, `community`, `spec`.
- `why` - чем источник полезен.
- `topics` - 3-8 тегов.
- `verified` - дата последней ручной проверки.

Для `index.yaml` дополнительно фиксируй:

- `priority` - насколько часто и рано агенту стоит брать этот файл в работу: `high`, `medium`, `low`.
- `scope` - тип материала: `core`, `advanced`, `operations`, `security`, `workflow`, `ecosystem`.
- `stability` - насколько быстро устаревает содержимое: `high`, `medium`, `low`.
- `trust` - основной уровень доверия к материалу: `official`, `spec`, `reference`, `community`, `mixed`.

## Как агенту читать metadata

- `priority: high` - читать раньше остальных по этой теме.
- `scope: core` - базовый слой, почти всегда полезен.
- `scope: advanced` - подключать, когда базовый слой уже понятен.
- `scope: operations` - использовать для деплоя, эксплуатации и инцидентов.
- `scope: security` - поднимать при auth, secrets, network policy и hardening задачах.
- `scope: workflow` - использовать для процессов разработки и работы с агентами.
- `stability: high` - можно чаще опираться без срочной перепроверки.
- `stability: medium/low` - лучше перепроверять перед критичными решениями.
- `trust: spec` и `trust: official` - приоритетные источники истины.
- `trust: community` - использовать как вспомогательный слой, а не как финальный authority.

## Минимальный workflow

1. Добавь ссылку в профильный `.md` файл.
2. Обнови `index.yaml`.
3. Если источник спорный, объясни в `why`, почему он здесь.
4. Если ссылка устаревает быстро, перепроверь ее перед использованием.
5. Для новой записи сразу выставляй `priority`, `scope`, `stability`, `trust`.

## Проверки

- Проверка структуры индекса:
  - `python scripts/validate_index.py`
- Строгая проверка покрытия markdown-файлов индексом:
  - `python scripts/validate_index.py --strict`
- Проверка ссылок:
  - `python scripts/check_links.py`
